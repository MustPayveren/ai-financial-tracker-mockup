# AI-Powered Financial Tracker — Web App Mock-Up

## Project Summary
This project is a working mock-up of an AI-powered Financial Tracker web application designed to help users understand and manage their spending. The app allows users to add income and expense transactions, view monthly summaries, and receive simple “AI insights” based on spending patterns. This mock-up focuses on demonstrating the user flow and core functionality without a real backend.

## Implemented Screens
### 1) Welcome / Login (index.html)
- **Purpose:** Start the user flow with a simple sign-in experience.
- **Simulated functionality:** No real authentication. The user’s email is saved to `localStorage` as a stub session and the user is redirected to the Dashboard.

### 2) Dashboard (dashboard.html)
- **Purpose:** Main overview of the user’s finances.
- **What it shows:**
  - Monthly summary cards: **Income**, **Expenses**, **Remaining**
  - **Recent transactions** list
  - **AI Insights** panel (rule-based insights)
- **Simulated functionality:**
  - Transactions are stored locally in `localStorage` (simulating database persistence).
  - Insights are generated by analyzing stored transactions (not a real ML model).
  - “Quick actions” allow clearing data, reseeding demo data, and exporting JSON (stub) to demonstrate app control and usability.

### 3) Add Transaction (add.html)
- **Purpose:** Enter new income/expense records.
- **What it includes:**
  - Type (income/expense)
  - Category (dynamic list based on type)
  - Amount, date, note
- **Simulated functionality:**
  - Form validation prevents empty/invalid inputs.
  - On save, the transaction is added to `localStorage` and the user is redirected to the Dashboard.

### 4) Reports (reports.html)
- **Purpose:** Summarize spending by category.
- **What it shows:**
  - Category totals for expenses in the selected month
  - AI insight summary based on the top spending category
  - Month switch controls (This month / Last month)
- **Simulated functionality:**
  - Report totals are computed from `localStorage` transactions.
  - Insight refresh demonstrates interactive updates.

## Data and “AI” Simulation
- **Database simulation:** Instead of a relational database, the mock-up uses browser `localStorage` to persist transactions. This keeps the project frontend-only while still demonstrating realistic data storage behavior.
- **AI simulation (rule-based):**
  - Calculates monthly totals and spending by category.
  - Identifies the top spending category.
  - Generates short suggestions (e.g., reduce food spending with meal prep).
  - Detects over-budget situations (expenses > income) and provides a warning message.

## User Flow (End-to-End)
1. User opens the app and signs in (stub).
2. User lands on the Dashboard and sees summary + insights.
3. User navigates to “Add Transaction” and submits a new entry.
4. User returns to Dashboard where totals, recent list, and insights update.
5. User opens Reports to view category totals and switches months to explore history.

## Notes / Future Improvements
- Replace `localStorage` with a real backend and a relational database (e.g., MySQL/PostgreSQL).
- Add secure authentication (user accounts).
- Expand AI features (trend detection, budgeting goals, predictions).
- Add charts/visualizations (spending over time).
